Types

CallFail

Represents calling failure.

Parent types

  • Error - Represents an error. Usually more specific error types are used.

Direct children types

  • ArgsMismatch - Represents calling failure due to arguments vs parameters mismatch.
  • DontKnowHowToCall - Represents calling failure when it is not known how to call the given object.
  • ImplNotFound - Represents calling failure when arguments do not match any method implementation.
  • StackDepthFail - Represents stack overflow error.
